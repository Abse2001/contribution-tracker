# Changelog 2026-04

- Copper pours can now be flagged as **unbroken** for contiguous fill control ([props #628](https://github.com/tscircuit/props/pull/628))
- Massive PCB-courtyard overhaul: switched to generic body-based / stepped outlines and fixed dimensions across SOIC, QFP, BGA, SOT*, JST, TO-220, VSSOP, Axial, etc. ([footprinter #584](https://github.com/tscircuit/footprinter/pull/584), [#585](https://github.com/tscircuit/footprinter/pull/585), [#586](https://github.com/tscircuit/footprinter/pull/586), [#587](https://github.com/tscircuit/footprinter/pull/587), [#588](https://github.com/tscircuit/footprinter/pull/588), [#589](https://github.com/tscircuit/footprinter/pull/589), [#590](https://github.com/tscircuit/footprinter/pull/590), [#591](https://github.com/tscircuit/footprinter/pull/591), [#593](https://github.com/tscircuit/footprinter/pull/593), [#594](https://github.com/tscircuit/footprinter/pull/594))
- EasyEDA importer improvements: preserves `{NAME}` silkscreen placeholders, supports oval plated holes, and handles undefined tokens ([easyeda-converter #374](https://github.com/tscircuit/easyeda-converter/pull/374), [#377](https://github.com/tscircuit/easyeda-converter/pull/377), [#376](https://github.com/tscircuit/easyeda-converter/pull/376))
- Autorouter reliability fixes: route-stitching bug resolved and copper-pour obstacles recognised ([autorouter #842](https://github.com/tscircuit/tscircuit-autorouter/pull/842), [#840](https://github.com/tscircuit/tscircuit-autorouter/pull/840))
- Autorouter now enforces via-to-copper clearance and can output stage-by-stage PNG debug dumps ([autorouter #854](https://github.com/tscircuit/tscircuit-autorouter/pull/854), [#848](https://github.com/tscircuit/tscircuit-autorouter/pull/848))
- Drag-and-drop 3D model loading added to CAD viewer landing page ([cad-component-viz #5](https://github.com/tscircuit/cad-component-viz/pull/5))
- PCB Viewer UX: toolbar overlay no longer captures canvas gestures, drill layer always on top, and errors are grouped/collapsed ([pcb-viewer #729](https://github.com/tscircuit/pcb-viewer/pull/729), [#735](https://github.com/tscircuit/pcb-viewer/pull/735), [#733](https://github.com/tscircuit/pcb-viewer/pull/733))
- SVG exporter: PCB courtyards render with round line-joins and correct primitive layering ([circuit-to-svg #538](https://github.com/tscircuit/circuit-to-svg/pull/538), [#543](https://github.com/tscircuit/circuit-to-svg/pull/543))
- KiCad → CircuitJSON: added inner-layer support, arc parsing, and accurate solder-paste / drill alignment ([kicad-to-circuit-json #54](https://github.com/tscircuit/kicad-to-circuit-json/pull/54), [#53](https://github.com/tscircuit/kicad-to-circuit-json/pull/53), [#58](https://github.com/tscircuit/kicad-to-circuit-json/pull/58), [#59](https://github.com/tscircuit/kicad-to-circuit-json/pull/59))
- Missing KiCad 3D models now fall back to STEP files automatically ([kicad-mod-cache #21](https://github.com/tscircuit/kicad-mod-cache/pull/21))
- STEP export gains face-level colours and style reuse for smaller files ([circuit-json-to-step #71](https://github.com/tscircuit/circuit-json-to-step/pull/71))
- New `tsci check trace-length` CLI command validates maximum trace runs ([cli #2633](https://github.com/tscircuit/cli/pull/2633))
- USB-C inner schematic symbol added and flipped-component courtyard rotation fixed ([core #2118](https://github.com/tscircuit/core/pull/2118), [#2088](https://github.com/tscircuit/core/pull/2088))
- BOM CSV export now includes footprints from embedded CAD models ([circuit-json-to-bom-csv #10](https://github.com/tscircuit/circuit-json-to-bom-csv/pull/10))
